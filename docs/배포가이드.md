# 배포 가이드

## GitHub 레포지토리에 업로드

### 1. Git 초기화 및 커밋

```bash
# Git 저장소 초기화
git init

# 모든 파일 추가
git add .

# 첫 커밋
git commit -m "Initial commit: SF 리마스터 웹사이트 구축"

# 브랜치명을 main으로 설정 (GitHub 기본값)
git branch -M main
```

### 2. GitHub 레포지토리 생성

1. [GitHub](https://github.com)에 로그인
2. 새 레포지토리 생성 (New Repository)
3. 레포지토리 이름 입력 (예: `sf-remaster-website`)
4. Public 또는 Private 선택
5. **README, .gitignore, license는 추가하지 않음** (이미 프로젝트에 있음)
6. "Create repository" 클릭

### 3. 원격 저장소 연결 및 푸시

```bash
# 원격 저장소 추가 (YOUR_USERNAME과 REPO_NAME을 실제 값으로 변경)
git remote add origin https://github.com/YOUR_USERNAME/REPO_NAME.git

# 코드 푸시
git push -u origin main
```

## Vercel 배포

### 방법 1: GitHub 연동 (권장)

1. [Vercel](https://vercel.com)에 로그인 (GitHub 계정으로)
2. "Add New Project" 클릭
3. GitHub 레포지토리 선택
4. 프로젝트 설정:
   - **Framework Preset**: Other
   - **Root Directory**: `.` (기본값)
   - **Build Command**: (비워두기)
   - **Output Directory**: `src/html`
   - **Install Command**: (비워두기)
5. "Deploy" 클릭

### 방법 2: Vercel CLI 사용

```bash
# Vercel CLI 설치
npm i -g vercel

# 프로젝트 루트에서 배포
vercel

# 프로덕션 배포
vercel --prod
```

### Vercel 설정 확인

프로젝트에 `vercel.json` 파일이 포함되어 있습니다. 이 파일이 자동으로 인식되어 배포 설정이 적용됩니다.

## 배포 후 확인

배포가 완료되면 Vercel에서 제공하는 URL로 사이트에 접속할 수 있습니다.

### 환경 변수 (필요시)

현재 프로젝트는 환경 변수가 필요하지 않지만, 나중에 API 키 등을 사용할 경우 Vercel 대시보드에서 설정할 수 있습니다.

## 주의사항

- 이미지 경로가 상대 경로(`../../assets/`)로 설정되어 있으므로, 실제 이미지 파일이 `assets/` 폴더에 있어야 합니다.
- PDF 파일은 크기가 크므로 `.gitignore`에 추가하거나 Git LFS를 사용하는 것을 권장합니다.
